# 南京线体项目开发规则 (guize.md)

## 1. 基础开发规则 (General)
- **回复语言**：始终使用 **中文** 进行交流和代码注释。
- **确认机制**：在删除文件或目录、以及安装新插件前，必须询问并获得许可。

## 2. 后端开发规范 (C# / .NET)
- **命名规范**：遵循 `PascalCase`（大驼峰）命名，如 `TorqueControllerService`。
- **异步编程**：必须使用 `async/await` 异步编程，防止界面或硬件通信阻塞。
- **通信枢纽**：SignalR Hub 方法应保持精简，复杂报文解析应放在 `Services` 层处理。

## 3. 前端开发规范 (Vue 3)
- **API 风格**：统一使用 Vue 3 `<script setup>` 组合式 API。
- **类型安全**：
  - 优先使用 TypeScript 声明类型，严禁使用 `any`。
  - MES 接口返回的数据结构必须定义对应的 `interface`。
- **样式指南**：
  - 推荐使用 Tailwind CSS 进行页面布局。
  - 保持深色工业科技风格，遵循 `1920x1080` 的产线大屏兼容性。

## 4. 报错处理与调试
- **日志分析**：如果代码运行报错，请先分析运行日志，不要直接重写整个文件。
- **仿真测试**：在修改核心业务逻辑时，需确保 `MOCK-` 仿真模式能够同步正常运行。

## 5. 业务安全规则
- **扫码枪处理**：条码扫码需在 `handleScan` 中进行修剪（Trim）和异常校验。
- **资源清理**：在组件卸载时（onUnmounted），必须清理所有激活的定时器、SignalR 监听及 TCP 临时连接。

- **文档即代码**：保持逻辑与文档（如 `development_log.md`）同步更新，让代码库永远处于可追溯、可审计的状态。
